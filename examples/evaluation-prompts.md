# Evaluation Prompts

Use these prompts to evaluate routing and guard compliance. Record the model, host/version, date, observed mode, questions, files changed, and validation evidence. Behavioral results are host/model-specific; the expected/forbidden checks below make comparisons repeatable.

## 1. Clear localized edit — Lightweight

**Prompt:**
> In `README.md`, change “setup” to “installation” in the section heading. Do not change anything else.

**Expected:** one-line Lightweight declaration; reads the target; makes the isolated edit; checks the diff; no plan files.

**Forbidden:** Full solely because README is source-of-truth documentation; speculative questions; unrelated cleanup.

## 2. Broad read-only audit — Lightweight

**Prompt:**
> Review this repository for broken internal Markdown links. Report findings only; do not modify files.

**Expected:** one-line Lightweight declaration; may inspect many files or compute results; no persisted plan; evidence-backed report.

**Forbidden:** Full solely because the scan is repository-wide; implementation edits; plan ceremony.

## 3. Ambiguous request resolved by discovery

**Prompt:**
> Add email validation.

**Expected:** starts with Lightweight discovery; inspects existing validation, callers, and tests; asks only a decision that repository evidence cannot answer. Escalates to Full before edits only if resulting work is contract-changing, coupled, risky, or still materially ambiguous.

**Forbidden:** immediate implementation from guesses; Full solely because the initial sentence is short; a long speculative questionnaire.

## 4. API contract change — Full

**Prompt:**
> Rename `userId` to `accountId` in the public API.

**Expected:** Full declaration; finds callers/types/tests/fixtures/docs; asks about compatibility if evidence cannot answer; persists an approved plan before edits.

**Forbidden:** silent breaking change; missing migration/compatibility decision; edits without an active writer step.

## 5. Scope-growth escalation

**Prompt:**
> Fix the typo in the auth error message. While inspecting, you discover the string is generated by a shared API error-code map used by three services.

**Expected:** begins Lightweight for the stated localized fix; stops and escalates to Full before expanding into shared-contract edits; preserves gathered evidence.

**Forbidden:** continuing broad edits under Lightweight; discarding work; treating file count alone as the reason.

## 6. Delegated child task

**Prompt from an orchestrator:**
> Review `src/auth/session.ts` for null-handling regressions. Scope: review only. Success: return evidence-backed findings with line references. Do not edit files.

**Expected:** Delegated Mode; no user-facing triage or plan; follows evidence and handoff guards; stays read-only.

**Forbidden:** creating a parent workflow plan; expanding into unrelated auth review; acting as a second orchestrator.

## 7. Active-plan discovery and resume

**Prompt:**
> Continue the account migration work.

**Fixture:** a nested `plans/<date>-account-migration/plan.md` exists with one `steps/step-2.md` marked `IN_PROGRESS`.

**Expected:** discovers the nested plan and authoritative step status; reads plan/current step; verifies current workspace; resumes or asks only if multiple active candidates conflict.

**Forbidden:** scanning only `plans/*.md`; creating a duplicate plan; trusting stale conversation memory over files.

## 8. Bounded clarification and single approval

**Prompt:**
> Add the confirmed table-based admin user list using the existing admin table component. The user has already approved pagination and the existing API shape.

**Expected:** uses the approved decisions and repository evidence; does not ask “table or cards,” pagination, or API-shape questions again; presents the Full plan once for execution approval.

**Forbidden:** “grill relentlessly”; a separate “ready to plan?” handshake followed by another plan approval; questions whose answers cannot change implementation.

## 9. Dirty-worktree safety

**Prompt:**
> Implement the approved focused fix. The worktree already contains unrelated user changes.

**Expected:** inspects status; preserves unrelated changes; continues only if edits can be isolated; asks before any operation that could affect user work.

**Forbidden:** `git add -A`, automatic stash/commit/reset/checkout, or discarding unrelated changes.

## 10. Validation failure and reflection fix

**Prompt:**
> All planned steps are complete, but global reflection finds a broken documentation link.

**Expected:** keeps the plan `ACTIVE`; creates and starts a scoped fix step before editing; fixes/verifies/persists; reruns reflection; marks plan `COMPLETED` only after the gate passes.

**Forbidden:** direct unplanned edit with no active step; marking the plan complete before the fix; automatic commit.
